Secrets Backends

  • Are "roots" of a filesystem
  • Different types of backends for different types of secrets
  • Types: generic, aws, cubbyhole, and more

Authentication: Tokens

  • Vault has no sense of users, only tokens
  • Everybody authenticates with tokens
  • $ vault auth d08e2bd5-ffb0-440d-6486-b8f650ec8c0c
  • Once authenticated, tokens are stored locally
  • Tokens are automatically included in commands that require permissions
  • Tokens are assigned permissions

Authentication Backends

  • Other methods to generate tokens
  • Entry points to get tokens
      • Examples
    • Github backend
    • Userpass backend
    • Token backend
    • TLS certificates
    • many others

Userpass

  • Use a username and password to generate a token
$ vault auth -methods=userpass \
     username=foo \
     password=bar
  • Permissions are set to userpass entries instead of tokens

TTL & Max TTL

  • Time to Live
  • Tokens and secrets have ttl's
  • Static secrets don't depend on ttl's. They exist until deleted
  • Dynamic secrets and tokens are deleted at the end of their ttl
  • TTL's can be renewed until Max TTL is reached
    Example for userpass
  • Userpass tokens with TTL = Max_TTL = 8 hours
  • Generated tokens last a "work day"

User permissions

  • Read
  • Update
  • List
  • sudo

path "secret/*" {
   capabilities = ["read", "list", "create", "update"]
}

path "secret/foo" {
   capabilities = ["read"]
}

path "secret/super-secret" {
   capabilities = ["deny"]
}
